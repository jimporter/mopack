Command-Line Reference§

Global options§

-h, --help§

Print a help message and exit. Equivalent to the help subcommand.

--verbose§

Show verbose output, e.g. the output of build commands during mopack resolve.

-c, --color WHEN§

Show colored output; WHEN is one of always, never, or auto and defaults to auto (i.e. show colored output when the terminal is a tty). -c is equivalent to --color=always.

--warn-once§

Only emit a given warning once.

Sub-commands§

mopack help [SUBCOMMAND]§

Print a help message and exit. If SUBCOMMAND is specified, print help for that subcommand.

mopack resolve [FILE]§

Fetch dependencies from their origins and prepare them for use by the current project (e.g. by building them).

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

-P TYPE=PATH, --deploy-path TYPE=PATH§

Set the directory to deploy package data type TYPE to PATH. TYPE is a GNU-like directory variable, such as prefix or bindir.

-o OPTION=VALUE, --option OPTION=VALUE§

Override the common option OPTION to be VALUE.

-S OPTION=VALUE, --source-option OPTION=VALUE§

Override the source option OPTION to be VALUE.

-B OPTION=VALUE, --builder-option OPTION=VALUE§

Override the builder option OPTION to be VALUE.

--strict§

Return an error during mopack usage if the requested dependency is not defined.

mopack usage [DEPENDENCY]§

Retrieve information about how to use a dependency. This returns metadata in YAML format (or JSON if --json is passed) pointing to a pkg-config .pc file.

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

--json§

Display usage results as JSON.

--strict§

Return an error if the requested dependency is not defined.

mopack deploy§

Copy the project's dependencies to an installation directory (e.g. as part of running a command like make install).

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

mopack clean§

Clean the mopack package directory of all files.

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

mopack list-files§

List all the input files used by the current configuration.

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

--I, --include-implicit§

Include implicit input files.

--json§

Display results as JSON.

--strict§

Return an error if the package directory does not exist.

mopack list-packages, mopack ls§

List all the package dependencies.

--directory PATH§

The directory storing the local package data; defaults to ./mopack.

--flat§

List packages without hierarchy.

mopack generate-completion§

Generate shell-completion functions for mopack and write them to standard output. This requires the Python package shtab.

-s SHELL, --shell SHELL§

Specify the shell to generate completion for, e.g. bash. Defaults to the current shell's name.
